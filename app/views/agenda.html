Since AOSP is a huge Open Source project, and around it you can find a lot of Open Source libraries/code, in this talk I will speak about what you can and can't do with Open Source Software, and try to answer some questions like:

  • Is Open Source the same than Free Software?
  • All the code we found on internet and we can use is Open Source?
  • Can I use X license with my proprietary app?
  • Can I mix X and Y licenses?
  • Can I make money if I use or develop open source software? (business models)
  • How can I contribute to an Open Source Software project?
  • What are the benefits of release my app under open source software? and the risks?

So this talk would try to clarify all the Open Source related stuff you can deal with in your apps.

In the creation of Wunderlist we have learned many things.

Since June 2012 when we started the creation of the 2nd release of Wunderlist, we have growth from around 400.000 downloads to more than 2 millions, we’ve been featured in Google Play Store (actually we are in the Editor’s Choice), Google Play for Education, Amazon App Store, etc.

Now with the release of our last version in late July of this year, we have taken Wunderlist beyond than never, we have changed the whole architecture and core of the app to make it more stable, faster and easier to maintain. Wunderlist has now a much better design, real time synchronization, and new and great features.

So in this talk I will talk about the things we’ve learned building a multimillion and successful Android app as Wunderlist is.

Uno de los objetivos o planes de futuro más conocido en gran parte de la comunidad de desarrolladores es crear su propio proyecto/aplicación/servicio. Pero la mayoría de ellos tan solo disponen de recursos y conocimientos limitados.

En esta charla se abordará este tema desde el punto de vista técnico y de producto, en un entorno de escasez de recursos tanto económicos como en conocimientos. Se tratará de reducir las necesidades al mínimo viable y se tomará ventaja de sistemas como Android y plataformas como App Engine, bajo una filosofía de desarrollo ágil, basada en vivir en beta y tratando de recoger feedback constante de agentes externos.

Estos conceptos se aplicarán sobre un caso práctico: una aplicación Android sustentada sobre una Api construida en App Engine (no-sql datastore o search api) en Python.

Google Cloud Platform

Cloud Endpoints es una de las últimas incorporaciones a la familia de Google Cloud Platform. Con Cloud Endpoints podrás crear tus propios REST APIs simplemente añadiendo anotaciones a tu código y generar librerías cliente para Android, iOS y JavaScript, manejando credenciales OAuth2 transparentemente.

En esta charla Nacho Coloma (Google Developer Expert para la plataforma Cloud) introducirá la forma de desarrollar con endpoints, y sus principales puntos fuertes y débiles.
